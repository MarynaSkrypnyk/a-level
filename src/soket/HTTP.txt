

Структура GET:
GET / HTTP/1.1
Host: tata.com
User-Agent: firefox/5.0 (Linux; Debian 5.0.8; en-US; rv:1.8.1.7)

GET — метод запроса;
/ — путь запроса (path);
HTTP/1.1 — версия протокола передачи данных.
Затем следуют заголовки:
Host — хост, которому адресован запрос;
User-Agent — клиент, который отправляет запрос.
Тело сообщения отсутствует.

Сообщения, отправленные на сервер, содержат только URL-адрес.

Структура POST сразу с примером JSON:

POST /user/create/json HTTP/1.1
Accept: application/json (аутентификация - список MIME типов, которые ожидает клиент.)
Content-Type: application/json (позволяет клиенту определить MIME тип документа.)
Content-Length: 200 (указывает размер отправленного получателю тела объекта в байтах.)
Host: tata.com (хост которому адресован запрос)

{
  "Id": 12345,   (тело)
  "User": "Ben"
}
Сообщения помещают любые необязательные параметры данных в тело сообщения запроса, а не добавляют их в конец URL-адреса.
